[
  {
    "name": "Senior Backend Engineer",
    "company": "Acme Corp",
    "url": "https://example.com/jobs/123",
    "salary": "€90,000 - €100,000/year",
    "description": "Full job description text...",
    "keywords": ["Python", "Flask", "MongoDB"],
    "state": "open"
  }
]

state is optional and defaults to open. A closed opportunity closes the matching job if one exists, otherwise it lands as a closed record kept for keyword analysis.

Pending URLs

Paste job posting URLs, one per line, to queue them for processing. Each is checked against imported jobs and staged records, then held here — out of the staging table below — until an MCP client retrieves, validates and imports it, at which point it becomes a viewable opportunity. URLs are reduced to their base (tracking parameters are stripped), and search-results pages are rejected. Queuing a URL marks the resulting opportunity Saved, putting it on your radar.
